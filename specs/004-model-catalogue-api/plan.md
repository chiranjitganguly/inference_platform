# Implementation Plan: Model Catalogue API

**Branch**: `004-model-catalogue-api` | **Date**: 2026-05-28 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `/specs/004-model-catalogue-api/spec.md`

## Summary

Expose `GET /v1/models` through the Kong gateway (port 8080) backed by LiteLLM's native model-list endpoint. All 11 platform models are defined in `services/litellm/config.yaml` with extended `model_info` metadata (tier, type, status, context_window, capabilities, dimensions). Kong enforces key-auth; unauthenticated requests are rejected with HTTP 401 before reaching LiteLLM.

## Technical Context

**Language/Version**: Python 3.11 (contract test only); YAML config files for LiteLLM and Kong

**Primary Dependencies**: LiteLLM main-v1.52.0 (model proxy), Kong 3.6 (gateway + key-auth)

**Storage**: N/A — read-only endpoint sourced from static YAML config

**Testing**: pytest 7.x (contract tests), bash (smoke-test.sh probe already exists)

**Target Platform**: Linux server / Docker Compose (OrbStack on Mac for dev)

**Project Type**: Platform API — extension of existing inference gateway

**Performance Goals**: p95 < 200 ms at the Kong gateway edge

**Constraints**: LiteLLM port 4000 must have no host binding; all traffic via Kong :8080

**Scale/Scope**: 11 models, read-only, no concurrency concerns beyond Kong connection pool

## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

| Principle | Status | Notes |
|---|---|---|
| I. Request Flow Integrity | **PASS (exemption noted)** | `GET /v1/models` is a read-only metadata query, not an inference request. Constitution §2.1 scopes the mandatory chain to "inference requests". Routing Kong → LiteLLM directly (bypassing Guardrails) is valid here — no prompt content, no PII, no content scanning required. |
| II. Prompt Content Ephemeral | **PASS** | No prompt or response content is involved. |
| III. OpenAI API Compatibility | **PASS** | Response follows OpenAI `/v1/models` envelope (`object: list`, `data: []`). Platform fields (`tier`, `type`, `status`, `context_window`, `capabilities`, `dimensions`) are additive — OpenAI SDK clients ignore unknown fields. |
| IV. Defence in Depth | **PASS** | Kong key-auth handles authentication (layer 1). OPA model-level ABAC is not required for a catalogue read — all models are visible to any valid key. Guardrails content scanning is not applicable to a metadata endpoint. |
| V. Falsifiable Acceptance Criteria | **PASS** | All SC-001 through SC-005 are curl-verifiable. Contract test validates schema. |
