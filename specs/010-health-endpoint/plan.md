# Implementation Plan: Platform Health Endpoint

**Branch**: `010-health-endpoint` | **Date**: 2026-05-30 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/010-health-endpoint/spec.md`

## Summary

Expose LiteLLM Proxy's built-in `GET /health` endpoint to external callers (load balancers, monitoring tools) by adding an unauthenticated Kong route at `/health`, and tune the existing Docker healthcheck parameters. No application code is required — LiteLLM already provides the endpoint; the work is purely configuration.

## Technical Context

**Language/Version**: N/A — configuration-only change (YAML)

**Primary Dependencies**: LiteLLM `main-v1.52.0` (built-in `/health`), Kong `3.6` (declarative config)

**Storage**: N/A — health response is computed on demand, nothing persisted

**Testing**: `curl` + `docker ps` for acceptance; Docker healthcheck for container-level liveness

**Target Platform**: Docker Compose `core` profile (Mac dev), GKE (production)

**Project Type**: Infrastructure configuration (gateway + container orchestration)

**Performance Goals**: `/health` responds in < 1s when all providers healthy; < 2s when any provider is timing out (per SC-001, SC-002)

**Constraints**: Port 4000 has no host binding — Docker healthcheck must use `localhost:4000` (internal to LiteLLM container). Kong is the only externally accessible port (8080).

**Scale/Scope**: Two file changes — `services/kong/kong.yml` and `docker-compose.yml`. Zero new containers.

## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

| Principle | Status | Notes |
|---|---|---|
| I — Request Flow Integrity | ✅ Pass | `/health` is a monitoring path, not an inference request. Constitution §2.1 covers inference requests exclusively. Docker healthcheck polls `localhost:4000` (internal). Kong still handles external access at :8080. |
| II — Prompt Content Ephemeral | ✅ Pass | Health endpoint contains no prompt or response content. |
| III — OpenAI Compatibility | ✅ Pass | Not an inference endpoint; OpenAI schema not required. |
| IV — Defence in Depth | ⚠️ Justified deviation | `key-auth` bypassed for `/health`. **Justification**: Load balancers and monitoring tools cannot authenticate. Endpoint is read-only, returns no sensitive data (no keys, no credentials, no prompt content). Standard industry exemption for liveness checks. |
| V — Falsifiable Acceptance Criteria | ✅ Pass | All acceptance tests expressed as `curl` commands with exact expected HTTP status and response fields. |

**Post-design re-check**: All principles remain satisfied. Configuration-only change introduces no new architectural risk.

### Kong Route Addition (`services/kong/kong.yml`)

Add inside the `litellm` service routes list:

```yaml
      - name: litellm-health
        paths:
          - /health
        methods:
          - GET
        strip_path: false
        # No key-auth plugin — unauthenticated access required for load balancers
```

### Docker Healthcheck Update (`docker-compose.yml`)

Replace the LiteLLM `healthcheck` block:

```yaml
    healthcheck:
      test: ["CMD-SHELL", "curl -sf http://localhost:4000/health || exit 1"]
      interval: 30s
      timeout: 10s
      retries: 5
      start_period: 20s
```

### Acceptance Criteria (Falsifiable)

| Criterion | Test | Expected |
|---|---|---|
| SC-001: responds < 1s (healthy) | `curl -o /dev/null -w "%{time_total}" http://localhost:8080/health` | `< 1.000` |
| SC-002: responds < 2s (any timeout) | Same command with one provider down | `< 2.000` |
| SC-003: no auth required | `curl -s http://localhost:8080/health \| jq .status` | `"healthy"` or `"unhealthy"` (no 401) |
| SC-003: auth header ignored | `curl -s -H "Authorization: Bearer bad" http://localhost:8080/health \| jq .status` | `"healthy"` or `"unhealthy"` (no 401) |
| SC-005: status field honest | Stop all providers; poll `/health`; check `status` | `"unhealthy"` |
| Docker healthcheck | `docker ps --format "{{.Status}}" \| grep litellm` | `(healthy)` |
| HTTP always 200 | `curl -o /dev/null -w "%{http_code}" http://localhost:8080/health` | `200` |
| Memory within budget | `make stats` with `core` profile | Total ≤ 620 MB |
