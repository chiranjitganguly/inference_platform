Each feature card below contains three blocks — one per SpecKit command step. Open a feature, copy the green block first, paste it into your AI agent chat. When the agent finishes that step, copy the next block. Never paste all three at once.

/speckit.specify Paste first. Describes WHAT to build and WHY. No tech stack mentioned. Agent creates the feature branch and spec.md file.
/speckit.clarify Paste second. Business rules, constraints, acceptance criteria, and edge case handling. Agent updates spec.md and runs /speckit.checklist.
/speckit.plan Paste third. Tech stack, library versions, Docker images, and implementation approach. Agent creates plan.md and the task list.
Badge legend:  new feature added in Phoenix/Langfuse update   updated existing feature revised to include Phoenix/Langfuse integration
Recommended sequence: work one phase at a time, implement all features in a phase before moving to the next. Run /speckit.constitution once at the start to define platform principles before your first feature.
